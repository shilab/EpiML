1. Introduction

Epistasis describes a phenomenon that different genetic variants affect phenotypes jointly instead of independently. Epistasis among different variants contributes to a large proportion of the hidden in heritability of complex traits [1-3], and plays an important role in gene expression and regulation [4-6]. However, genomic data is typically high-dimensional, where the number of features (genetic variants) is usually much larger than sample size. The large number of genetic variants that present in high dimensional genomic data make it extremely challenging for epistasis analysis since the searching space for epistasis grows exponentially with the number of genetic variants. Penalized regression based machine learning methods work well with analyzing high-dimensional and large-scale genomic data in that they can use penalized terms to impose sparsity to simultaneously enable scalable feature selection and dimension reduction to avoid overfitting. Such methods including classical Least Absolute Shrinkage and Selection Operator (LASSO) [7], newly proposed spike-and-slab LASSO generalized linear models (ssLASSO GLMs) [8, 9] and Empirical Bayesian Elastic Net (EBEN) [10, 11] are proved to well scalable the large dimension genomic data in recent studies. Nonetheless, these methods usually include multiple hyperparameters that need to be carefully trained for optimization on a particular genomic dataset. The intrinsic complexity of optimizing these models limits their wide adoption and makes it hard to reproduce results.

To support reproducible epistasis analysis on high dimensional genomic datasets, we have developed an integrated and user-friendly web server for epistasis analysis using classical and emerging penalized machine learning methods (EpiML), which has many advantages: 1) it is easy to use. The users only need to upload their data, select a method and submit their job. Our server will handle all the processing in background. 2) It will provide many friendly interactive visualizations, like circle network, adjacent matrix and force-directed network. 3) For some case study, multiple external resources are integrated in results visualization for analysis. 4) User can customize their analysis. For each job, its corresponding Jupyter notebook is automatically generated. All notebooks can be downloaded, modified and rerun, allowing users to fully customize their epistasis analysis on local computers.

2. Getting Started

This is a step-by-step tutorial how to use EpiML to find out main and epistasis effects in your data.

2.1. Prepare your data

Input files include two parts: feature matrix X, and label matrix Y. Both of them are tab-delimited tables with row and column names. In generally, the row names and column names could be any string, but no repeat. However, for gene and microRNA case studies, the column names of feature matrix are very important, because they will be used for parsing the gene position for genotype and searching miRBase for microRNA.

The column names of genotype are combined with SNP id, chromosome id, position, reference allele, and alternative allele, separated by ‘_’. An example for genotype data format is shown in following:

The column names of microRNA are microRNA ID in miRBase. An example for microRNA data format is shown in following:

The format of label matrix Y for all kind of data are same. An example of label matrix Y is shown in following:

Important notes:

  • Only files with .txt or .csv extensions are accepted.
  • File size is limited to no more than 5MB.

2.2. Upload data

Some basic information is required before you upload data.

  • Job name is used to identify jobs when you retrieve results.
  • Email is used to accept notification and retrieve results. It is optional.
  • Job Category refers to what kind of job. For Gene and microRNA category, links to meta-data and database will be integrated in result visualization.
  • Data Type of Feature Matrix is used to preprocess feature matrix. For continuous data, it will be quantile normalized.
  • Description is used to help user annotate job details, such as purpose, type of data, and parameters. It is optional.
  • Upload the prepared files. Upload the feature matrix file and label matrix file separately.
  • Select Method.Three popular methods are built in web server. Refer to Machine Learning Methods section to learn more details. The cross validation is specifically set as 5.

2.3. Check job status and retrieve results

Once your job is submitted, it will be added into job queue waiting for execution in background. The browser will jump to a processing page where you can check your job status, as following figure shown. The results will be shown on your screen when it is finished. Because all jobs are running in background, you can bookmark and close this page. You also can reopen this page anytime to check job status. Meanwhile, if you provide an Email address, you will receive an Email notification with this page link.

3. Result and visualization

3.1. Summary

Once the submitted job is finished, you can download results through corresponding links, or directly check your results using following interactive tables and visualizations.

3.2. Interactive table

EpiML can detect significant main effects and epistasis effects. These detected results will be list in two interactive tables correspondingly, where you can rank whole table by anyone column and search all contents to find out your interesting features.

3.3. Visualization

We provides three kind of interactive visualizations, including circle network, adjacency matrix and force-directed graph. When you move mouse over one node, more detailed information and linked nodes will be highlighted.

Circle Network

One node in circle network represents a feature in detected effects. All nodes are arranged as a circle. Two nodes that have epistatic interaction are linked by an undirected edge. Circle network has two main advantages comparing to table: 1) It is easy to find out those features that have multiple epistatic effects with other features; 2) It is easy to find indirect relationship among features.

You can move mouse over one node to find its interaction nodes in red. When you click one node, the node and its associated nodes will be highlighted in green.

Adjacency Matrix

The row and column names in adjacency matrix both are the features in detected epistatic effects. Each cell \((i, j)\) represents an edge from vertex \(i\) to vertex \(j\). Matrix cell depicts coefficient value in epistasis effect table. The darker cells indicate strong interaction. Adjacency matrix has two advantages comparing to circle network: 1) Adjacency matrix can give a whole picture of epistatic relationship, especially for large number of epistatic features; 2) Matrix cell value can depict the strength of epistatic relationship.

You can move mouse over one node to check its epistatic feature and coefficient value.

Force-directed Graph

Force-directed graph is a more flexible way to represent epistatic relationship. One node in graph represents a feature in epistasis effect table. Two nodes that have epistatic relationship are linked by an edge. All kinds of information can be embedded in force-directed graph, like coefficient value and external resources.

You can move mouse over and drag one node to check its linked epistatic features. Click one node to show more information.

4. Customize Code

Users can customize analysis code based on released Jupyter notebooks and pre-configured Docker containers. For each job, its corresponding Jupyter notebook is automatically generated. All notebooks can be downloaded, modified and rerun based on pre-configured Docker container which contains a fully functional Jupyter server with all the Python and R libraries necessary to run the notebooks.

4.1. Install Docker

Docker is a Container-as-a-Service (CaaS) platform that enables a secure software supply chain and deploys diverse applications for high availability across disparate infrastructure. Download, install and run Docker on your computer from the official Docker website depending on your OS.

4.2. Generate job notebook

Submit your job on EpiML web server. Once the job is done, you can download it from job summary table. If your data is too large, you can submit the demo data following Get Started section.

4.3. Launch a Docker Container

  1. Open a new terminal window.
  2. Change work directory into that includes your job notebook. Strongly recommend to create a new directory, and copy notebook into the new directory.
  3. Launch the Docker container by entering the following command 
    docker run -it --rm -p 8888:8888 -v "$PWD":/home/jovyan/work workhardplayharder/epiml:latest
    This command pulls the workhardplayharder/epiml image tagged latest from Docker Hub if it is not already present on the local host. It then starts a container running a Jupyter Notebook server, and exposes the server on host port 8888. The server logs appear in the terminal.

4.4. Customize your Notebook

Copy/paste http://<hostname>:8888/?token=<token> in a browser loads the Jupyter Notebook dashboard page, where hostname is the name of the computer running docker and token is the secret token printed in the console. Click into work directory, and find the jupyter notebook. Customize and save your notebook in Jupyter. Once you are done, stop the Docker container by pressing Ctrl+C on the previously opened Terminal window.

5. Analysis Methods

We build-in three penalized regression based machine learning methods for epistatic analysis. In general, you only need to upload your data, select one machine learning method, and then submit your job. You don't need to care about the algorithm detail. Our server will handle all the procedure in backend. For your better understanding the workflow and difference among build-in methods, the build-in methods are briefly introduced.

The model for identifying main and epistasis effect is shown as below:

$$y= \mu + X \beta_{main} + (X_i \odot X_j) \beta_{epis} + \epsilon$$

$$i \neq j; i,j \in [1, p] $$

where \(y\) is the label, \(\mu\) is mean, \(X\) denotes the feature matrix with dimension \((n, p)\), \(X_i, X_j\) denotes the \(i^{th}\) and \(j^{th}\) features in the matrix \(X\). \(\odot\) denotes the symbol for elementwise multiplication of vectors. \(\beta_{main}\) denotes the main effects, \(\beta_{epis}\) denotes the epistatic effects of the \(i^{th}\) and \(j^{th}\) features, and \(\epsilon\) is the residual error which follows the normal distribution \(N(0,\sigma^2)\).

5.1. Workflow

Penalized regression based machine learning methods work well for high-dimensional data. However, they are designed for solving linear regression problem. We develop a workflow composed of multiple steps for main and epistasis detection, as shown in Figure 11.

  1. Identify candidate main effects from feature matrix. Run penalized regression based machine learning methods on feature matrix, by which some significant effects can be selected as candidate main effects.
  2. Calculate the phenotype of epistatic effect. The phenotype of epistatic effect can be calculated by removing those detected main effects from the original phenotype.
  3. Generate epistasis matrix. Generate epistasis matrix by pairwise features with element-wise multiplication.
  4. Identify candidate epistatic effects from epistasis matrix. Run penalized regression based machine learning methods on epistasis matrix, by which some significant effects can be selected as candidate epistatic effects.
  5. Identify significant main and epistatic effects. Rerun penalized regression based machine learning methods on the previously selected candidate main and epistatic effects to identify the final significant main and epistatic effects.

5.2. Methods

Three popular penalized regression based machine learning methods are used to in our web server to build the above workflow, including Least Absolute Shrinkage and Selection Operator (LASSO) [7], Spike-and-Slab LASSO generalized linear models (ssLASSO GLMs) [8, 9], and Empirical Bayesian Elastic Net (EBEN) [10, 11].

Least Absolute Shrinkage and Selection Operator (LASSO)

LASSO is one of the most commonly used method which introduces a \(L_1\) regularization term that adds a penalty term equivalent to absolute value of coefficients on all parameters. LASSO will randomly select one feature if a set of features are closely correlated. There is one hyperparameter \(\lambda\) to adjust the regularization degree which means it would induce more coefficients are shinked to zero if increasing \(\lambda\). Cross validation strategy is used to choose the \(\lambda\) to optimize the Lasso model [7].

Spike-and-Slab LASSO (ssLASSO)

ssLASSO GLMs imposes different shrinkage priors, spike-and-slab mixture double-exponential prior distribution, on related and unrelated parameters which can induce optimal model solutions. For the ssLASSO GLMs, \(y\) is represented by a link function that \(\eta = E(y_i|X_i)\). ssLasso GLMs combines the EM algorithm and cyclic coordinate descent algorithm to fastly fit high-dimension data. There are two hyperparameters (\(s_0\) and \(s_1\)) in ssLASSO algorithm. The \(s_0\) is a small value and serves as a “spike” scale for modelling irrelevant coefficients and inducing strong shrinkage, while \(s_1\) is chosen to be relatively large value and serves to be “slab” scale for modelling related coefficients and inducing no or weak shrinkage. The grid-search based method can be applied to select the optimal combination of two hyperparameters. In an alternative way, it’s reported that \(s_1\) has little influence on results comparing to \(s_0\), then users can use generalized linear model (GLM) to estimate the data scale to be as \(s_0\) [8, 9].

Empirical Bayesian Elastic Net (EBEN)

EBEN is a Bayesian based method which use a fast coordinate ascent method and doesn’t reply on MCMC procedure. Hence, EBEN can scalable the over-saturated genetic model to estimate the parameters in the model efficiently. In addition, EBEN can conduct the hypothesis test such as \(t\)-test using the covariance matrix and the coefficients which is different from elastic net method [5]. There are two hyperparameters, such as \(\lambda_1\) and \(\lambda_2\), are the components of a generalized Gamma distribution which imposes on the unknown variance of each coefficients. Cross-validation strategy is used to tune these two hyperparameters to optimize the EBEN model [10, 11].

6. FAQ

  1. How long does it take to get results?

    2. The processing time depends on the scale of upload data and selected method. In generally, the LASSO and ssLASSO are very fast. EBEN takes long time, especially number of folds is large. For hundreds of samples and features, LASSO and ssLASSO can be finished in several minutes. EBEN may need an hour.

  2. Why can't I upload files?

    3. First, please check your file extension. Only files with .txt or .csv extensions are accepted. Second, file size is limited to no more than 5MB due to time consuming. If your data is too large, please follow customize code section, and run the pipeline on local host.

  3. Which browser is recommended?

    4. EpiML is tested to work on the following browsers. We recommend using Chrome or Safari. If you are already using one of the above browsers try updating it. If it still doesn't work let us know.
